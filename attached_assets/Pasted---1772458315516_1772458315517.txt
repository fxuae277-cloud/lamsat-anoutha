تطوير صفحة “الجرد والتسويات” لمستوى محاسبي متقدم

المطلوب:
تطوير صفحة الجرد والتسويات بحيث تصبح “سجل تدقيق + أثر مالي” مثل الأنظمة المحاسبية الاحترافية، مع تحسين الأعمدة، الفلاتر، وحالات الجرد، وإضافة قيمة الفروقات، وتصدير تقرير.

1) تحسين الجدول الرئيسي (Inventory Counts List)
أ) الأعمدة الجديدة

عدّل جدول “عمليات الجرد” ليعرض الأعمدة التالية (بدلاً من الوضع الحالي أو مع الإبقاء على المهم):

رقم الجرد (مرجع فريد) مثل: JRD-000001

التاريخ + الوقت

الفرع

الموقع (المخزن/المعرض…)

أُنشئ بواسطة (اسم المستخدم)

عدد الأصناف

فرق زيادة (+qty)

فرق نقص (-qty)

قيمة الزيادة (+value) (محسوبة)

قيمة النقص (-value) (محسوبة)

صافي أثر الجرد (net value) (محسوب)

الحالة (مفتوح/مكتمل/مُعتمد/مُلغى)

إجراءات (عرض/طباعة/اعتماد/إلغاء)

ملاحظة: “القيمة” تعتمد على تكلفة الصنف (cost) وليس سعر البيع.

2) فلاتر بحث احترافية (Filters)

أضف أعلى الجدول شريط فلاتر:

نطاق العرض: كل الشركة / فرع محدد

الموقع

الحالة

المستخدم

تاريخ من/إلى

بحث (رقم الجرد أو اسم المستخدم)

مع زر:

تصدير تقرير (PDF/Excel)

3) حالات الجرد (Workflow)

غيّر الحالة من “جاري” فقط إلى:

open = مفتوح (يتم إدخال العد)

completed = مكتمل (تم إنهاء العد لكن لم يعتمد)

approved = مُعتمد (تم تثبيت الفروقات وتحديث المخزون)

cancelled = مُلغى

قواعد:

لا يتم تحديث المخزون فعلياً إلا عند approved

لا يسمح بالتعديل بعد approved إلا بصلاحية مدير

4) تفاصيل الجرد (صفحة View)

عند فتح عملية جرد (عرض التفاصيل) أضف:

أ) ملخص أعلى الصفحة

عدد الأصناف

إجمالي فرق الزيادة (qty/value)

إجمالي فرق النقص (qty/value)

صافي الأثر المالي

ب) جدول الأصناف داخل الجرد

الأعمدة:

الصنف

الباركود

الكمية بالنظام

الكمية المعدودة

الفرق (+/-)

تكلفة الوحدة (cost)

قيمة الفرق

5) الحسابات (Accounting Link) — أساسي الآن ومتقدم لاحقاً

عند اعتماد الجرد approved:

أنشئ سجل “حركة محاسبية” داخل النظام (حتى لو ليس دفتر أستاذ كامل الآن)

على الأقل خزّن:

inventory_adjustment_value

approved_by

approved_at

(لاحقاً يمكن تحويلها لقيد محاسبي: مخزون / فروقات مخزون / تكلفة مبيعات)

6) التقارير والطباعة

أضف زر داخل التفاصيل:

طباعة تقرير الجرد PDF
يتضمن:

رأس التقرير: رقم الجرد + التاريخ + الفرع + الموقع + المنشئ + المعتمد

ملخص الفروقات كمية/قيمة

جدول الأصناف

7) الصلاحيات (Permissions)

إنشاء جرد: موظف مخزن/مدير فرع

اعتماد جرد: مدير/مالك فقط

إلغاء جرد: مدير/مالك فقط

حذف: ممنوع (استبدله بـ Cancelled)

8) قاعدة البيانات (ملخص سريع لما يجب إضافته)

إضافة reference_code للجرد (مثل JRD-000001)

إضافة created_at timestamp

إضافة approved_at, approved_by

إضافة status enum

في تفاصيل الجرد:

system_qty, counted_qty, diff_qty

unit_cost, diff_value

النتيجة المتوقعة

صفحة الجرد تصبح:

قابلة للتدقيق

تعكس أثر مالي حقيقي

جاهزة لربطها بالمحاسبة لاحقاً

وتعمل بشكل ممتاز مع Multi-Branch

أريد منك تنفيذ إعادة هيكلة كاملة للـ Sidebar + الصلاحيات في مشروعنا (Node.js + React + Vite + Drizzle + PostgreSQL).

الهدف

المالك (OWNER) يرى “قائمة إدارة” كاملة.

الموظف (EMPLOYEE) يرى “قائمة تشغيل” مختصرة فقط تخص عمله.

الموظف لا يستطيع الدخول لأي صفحة تخص المالك حتى لو كتب الرابط يدويًا.

التغيير يكون دائم ومركزي: (Sidebar + حماية Routes + حماية APIs).

1) تعريف Roles

اعتمد هذين الدورين فقط:

OWNER

EMPLOYEE

وتأكد أن user.role موجود في جلسة الدخول ويرجع للعميل ضمن /api/user أو useAuth().

2) Sidebar واحد بمصدرين (OWNER/EMPLOYEE)

افتح الملف:
client/src/config/sidebar.tsx

واكتب/عدّل بحيث يحتوي:

OWNER_SIDEBAR

EMPLOYEE_SIDEBAR

ودالة getSidebarSections(role) لاختيار القائمة حسب الدور.

قائمة الموظف (EMPLOYEE) — تشغيل فقط

نقطة البيع (POS) => /pos

الطلبات => /orders

فواتير نقطة البيع => /pos-invoices

العملاء => /customers

تسجيل الخروج فقط

ممنوع يظهر له: المنتجات والأسعار، المخزون، الموردون، المشتريات، المصروفات، المحاسبة اليومية، التقارير، سجل العمليات، الإعدادات… إلخ

قائمة المالك (OWNER) — إدارة كاملة مختصرة ومتوازنة

قسمها كالتالي (بدون تكرار لوحات):
لوحة الإدارة (Dashboard)

لوحة التحكم => /dashboard

التشغيل (Operations)

نقطة البيع (POS) => /pos

الطلبات => /orders

فواتير نقطة البيع => /pos-invoices

البيانات (Master Data)

المنتجات والأسعار => /products

العملاء => /customers

المخزون (Inventory)

المخزون => /inventory

الجرد والتسويات => /stocktake

المشتريات (Purchasing)

الموردون والمشتريات => /suppliers-purchases

المرتجعات => /returns

المالية (Finance)

المصروفات => /expenses

المحاسبة اليومية => /daily-ledger

التقارير المالية => /financial-reports

الرقابة (Audit)

آخر العمليات => /activity-log

سجل المراجعة => /audit-log

الإعدادات (Settings)

الإعدادات => /settings

الرواتب والموظفين => /hr-payroll (إن كانت موجودة)

النسخ الاحتياطي => /backup

3) ربط Sidebar بالدور فعليًا (وليس فقط عرض)

افتح ملف الـ Sidebar Component الذي يعرض القائمة (ابحث عن: sidebarItems أو map(section.items)).
استبدل مصدر البيانات ليستخدم:
getSidebarSections(user.role)

لا أريد قوائم ثابتة بعد الآن.

4) حماية صفحات الواجهة (Front-end Route Guard)

نفّذ حارس صلاحيات:

أي Route خاص بالمالك ممنوع للموظف.

إذا موظف حاول يدخل رابط خاص بالمالك:

حوله إلى /pos أو صفحة “غير مصرح” مثل /unauthorized.

طبق الحماية في Router أو في Layout Guard (حسب هيكل المشروع).

5) حماية API (أهم شيء)

في السيرفر:

أضف Middleware requireRole("OWNER") و requireRole(["OWNER","EMPLOYEE"])

اربط كل Endpoints الحساسة للمالك فقط (مثل: المنتجات، الأسعار، المخزون، الموردين، المصروفات، التقارير، الإعدادات، النسخ الاحتياطي، سجلات التدقيق).

Endpoints التشغيل فقط (POS/Orders) تكون متاحة للموظف.

المطلوب: الموظف لا يستطيع تنفيذ أي عملية إدارية حتى لو نادى API مباشرة.

6) اختبار نهائي مطلوب منك

بعد التعديل:

ادخل بحساب OWNER: تظهر القائمة الكاملة وتعمل كل الروابط.

ادخل بحساب EMPLOYEE: تظهر القائمة المختصرة فقط.

جرّب فتح /settings أو /products كموظف: يجب منعها (Redirect أو Unauthorized).

جرّب نداء API إداري كموظف: يرجع 403.

7) لا تغيّر UI إلا الضروري

لا تغيّر تصميم الألوان أو شكل العناصر—فقط الهيكلة والصلاحيات.

ابدأ التنفيذ الآن، وإذا واجهت أي مسار Route غير موجود، ابحث عن المسارات الفعلية في مشروعنا وعدّلها لتطابق الموجود بدل اختراع مسارات جديدة.

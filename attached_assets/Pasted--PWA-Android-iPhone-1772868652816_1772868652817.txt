أريد إنشاء نسخة هاتف احترافية للمالك داخل النظام الحالي تعمل كتطبيق PWA على الهاتف (Android و iPhone) بدون نشر في المتاجر. هذه النسخة يجب أن تكون مخصصة للمالك وتحتوي على:

1) لوحة تحكم كاملة بالمؤشرات
2) إدخال فواتير المشتريات بالباركود
3) التحويل بين المخازن والفروع
4) الجرد من الهاتف

المطلوب بالتفصيل:

أولاً: لوحة التحكم للمالك على الهاتف
أنشئ Dashboard Mobile خاصة بالمالك وتحتوي على:

- مبيعات اليوم
- مبيعات الشهر
- عدد الفواتير
- متوسط الفاتورة
- الربح التقريبي
- المصروفات
- النقد الحالي
- عدد الفروع المفتوحة الآن
- عدد الشفتات المفتوحة
- المخزون المنخفض
- المنتجات النافدة
- التحويلات المعلقة
- الجرد غير المعتمد
- أفضل 5 منتجات مبيعاً
- أفضل فرع اليوم
- آخر العمليات في النظام

وأريد داخل اللوحة:
- فلتر: كل الفروع / فرع محدد
- فلتر: اليوم / 7 أيام / 30 يوم / هذا الشهر
- رسم بياني للمبيعات
- جدول مختصر لأداء الفروع
- قسم تنبيهات مهم
- قسم آخر العمليات
- أزرار سريعة:
  - فاتورة مشتريات جديدة
  - تحويل جديد
  - جرد جديد
  - مسح باركود

ثانياً: صفحة المشتريات من الهاتف
- اختيار المورد
- إدخال رقم فاتورة المورد
- رفع صورة الفاتورة
- زر مسح باركود المنتج بالكاميرا
- إضافة المنتج تلقائياً بعد قراءة الباركود
- إدخال الكمية
- إدخال سعر الشراء
- حساب إجمالي الفاتورة تلقائياً
- زر حفظ
- زر اعتماد الفاتورة وتحويلها إلى المخزون

ثالثاً: صفحة التحويل بين المخازن والفروع
- اختيار المصدر
- اختيار الوجهة
- مسح المنتجات بالباركود
- إدخال الكمية
- عرض المخزون المتوفر
- زر إرسال التحويل
- حالة التحويل: مرسل / مستلم / مكتمل

رابعاً: صفحة الجرد من الهاتف
- اختيار الفرع أو المخزن
- اختيار الفئة إن وجدت
- مسح المنتج بالباركود
- عرض الكمية المسجلة في النظام
- إدخال الكمية الفعلية
- حساب الفرق تلقائياً
- زر حفظ الجرد
- زر اعتماد التسوية

خامساً: تحويل النظام إلى PWA
- إضافة manifest.json
- إضافة service-worker.js
- دعم Add to Home Screen
- أيقونة للتطبيق
- واجهة Mobile Responsive
- عمل جزئي عند ضعف الإنترنت

سادساً: الصلاحيات
- فقط المالك يرى لوحة التحكم الكاملة وصفحات المشتريات والتحويل والجرد الخاصة بالمالك
- الموظف لا يرى هذه الصفحات
- حماية جميع APIs الخاصة بالمالك

سابعاً: تحسين تجربة الهاتف
- أزرار كبيرة
- حقول إدخال سهلة
- تصميم مناسب للشاشات الصغيرة
- شريط سفلي للتنقل:
  - الرئيسية
  - المشتريات
  - التحويل
  - الجرد
  - المزيد

ثامناً: APIs المطلوبة
- GET /api/mobile/owner/dashboard
- POST /api/purchases/mobile-create
- POST /api/transfers/mobile-create
- POST /api/inventory/mobile-count
- GET /api/products/barcode/:code

تاسعاً: الحماية والسجل
- السماح فقط للمالك بالوصول
- منع الموظفين
- تسجيل كل عملية في activity log

عاشراً: الاختبار
بعد التنفيذ اختبر فعلياً:
- لوحة التحكم على الهاتف
- مسح الباركود
- إدخال فاتورة مشتريات
- تحويل بين فرعين
- عملية جرد
- ظهور النتائج فوراً في لوحة المالك عند توفر الإنترنت

أريد التنفيذ داخل النظام الحالي وليس إنشاء مشروع جديد، ويكون التصميم احترافي وسريع ومناسب للهاتف.

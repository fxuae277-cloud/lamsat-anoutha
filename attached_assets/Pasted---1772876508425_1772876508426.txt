أريد العمل على النظام الحالي القائم لمسة أنوثة بصفته نظام إدارة ومحاسبة داخلي للشركة، ولكن مهم جداً:

1) لا أريد إعادة بناء الميزات الموجودة
2) لا أريد تكرار الصفحات أو APIs أو الجداول الموجودة
3) أريد أولاً مراجعة شاملة للنظام الحالي
4) ثم التأكد من أن كل ميزة موجودة تعمل فعلاً بشكل صحيح
5) وبعد ذلك فقط أضف الميزات غير الموجودة من النقاط المحاسبية والإدارية المطلوبة

اعمل داخل المشروع الحالي فقط، ولا تنشئ مشروعاً جديداً، ولا تنشئ مكونات مكررة إذا كان هناك شيء مشابه موجود.

==================================================
أولاً: مرحلة المراجعة قبل أي تطوير
==================================================

أريد منك أن تبدأ أولاً بعمل Audit / Review كامل للنظام الحالي، يشمل:

1) مراجعة الصفحات الحالية الموجودة فعلاً
راجع ما هو موجود حالياً في النظام مثل:
- لوحة التحكم
- نقطة البيع
- الطلبات
- فواتير البيع
- المنتجات والأسعار
- العملاء
- المخزون
- الجرد والتسويات
- الموردون والمشتريات
- المرتجعات
- المصروفات
- المحاسبة اليومية
- التقارير المالية
- آخر العمليات
- سجل المراجعة
- الإعدادات
- الرواتب والموظفين
- أي صفحات أخرى موجودة

2) مراجعة الوظائف الحالية
تأكد ما الموجود فعلاً وما حالته:
- هل يعمل بشكل كامل؟
- هل يعمل جزئياً؟
- هل موجود شكلياً فقط؟
- هل توجد بيانات ولكن بدون ربط صحيح؟
- هل توجد واجهة بدون Backend؟
- هل توجد APIs غير مكتملة؟
- هل توجد صفحة لكنها تحتاج تحسين فقط؟

3) مراجعة قاعدة البيانات والبنية
راجع:
- الجداول الحالية
- العلاقات بين الجداول
- الأعمدة المهمة
- هل يوجد تكرار غير ضروري؟
- هل توجد جداول موجودة يمكن إعادة استخدامها بدل إنشاء جديدة؟
- هل توجد أعمدة يجب استغلالها بدل بناء شيء إضافي

4) مراجعة APIs الحالية
راجع:
- ما APIs الموجودة حالياً
- ما الذي يعمل منها
- ما الذي يحتاج تعديل
- ما الذي يمكن إعادة استخدامه
- ما الذي ينقص فقط

==================================================
ثانياً: المطلوب في نتيجة المراجعة
==================================================

أريد منك قبل التطوير أن تخرج تقريراً داخلياً واضحاً (ثم تطبق بناءً عليه) بهذا التقسيم:

1) Features موجودة وتعمل جيداً
2) Features موجودة لكن تحتاج إصلاح
3) Features موجودة لكن تحتاج تحسين واجهة أو ربط
4) Features غير موجودة فعلاً وتحتاج إضافة
5) Features مكررة أو يمكن دمجها
6) APIs موجودة يمكن إعادة استخدامها
7) جداول موجودة يجب استخدامها بدل إنشاء جداول جديدة

المهم:
- لا تضف شيئاً جديداً إذا كان موجوداً ويمكن إصلاحه أو تحسينه
- استخدم الموجود أولاً
- أصلح وراجع قبل البناء

==================================================
ثالثاً: الميزات التي أريد مراجعتها أولاً قبل إضافتها
==================================================

أريد مراجعة هذه النقاط المحاسبية والإدارية بالتحديد، والتأكد هل هي موجودة أم لا:

1) دليل الحسابات (Chart of Accounts)
2) القيود المحاسبية (Journal Entries)
3) دفتر الأستاذ (General Ledger)
4) كشف حساب العملاء
5) كشف حساب الموردين
6) إدارة أرصدة العملاء والموردين
7) تنبيهات المخزون والحد الأدنى
8) تقرير الأرباح والخسائر
9) تقارير حركة المخزون
10) Dashboard ذكية بتحليلات أفضل
11) النسخ الاحتياطي التلقائي واليدوي
12) صلاحيات المستخدمين المتقدمة

لكل نقطة من هذه النقاط:
- تحقق هل هي موجودة بالكامل
- أو موجودة جزئياً
- أو غير موجودة
- أو موجودة ولكن تحتاج تطوير فقط

==================================================
رابعاً: منهج التنفيذ المطلوب
==================================================

أريد منك العمل بالترتيب التالي:

المرحلة 1:
- مراجعة الموجود
- منع التكرار
- إعادة استخدام الموجود
- إصلاح ما هو معطل أو ناقص

المرحلة 2:
- تحسين ما هو موجود لكن غير مكتمل
- إكمال الربط بين Frontend و Backend
- إكمال APIs الناقصة
- إصلاح الحسابات والتقارير التي لا تعمل كما يجب

المرحلة 3:
- إضافة فقط الميزات غير الموجودة فعلاً
- بشرط ألا تكون مكررة مع شيء حالي

المرحلة 4:
- اختبار فعلي
- التأكد أن كل شيء يعمل
- التأكد أن الإضافات الجديدة لا كسرت الموجود

==================================================
خامساً: أولويات الإضافة بعد المراجعة
==================================================

إذا تبين أن بعض الميزات غير موجودة فعلاً، فأضفها بهذا الترتيب:

الأولوية 1:
- كشف حساب العملاء
- كشف حساب الموردين
- تنبيهات المخزون
- النسخة الاحتياطية

الأولوية 2:
- تقرير الأرباح والخسائر
- تقارير حركة المخزون
- أرصدة العملاء والموردين

الأولوية 3:
- دليل الحسابات
- القيود المحاسبية
- دفتر الأستاذ
- صلاحيات متقدمة
- Dashboard أذكى

لكن لا تبدأ بإضافة أي واحدة قبل التأكد أنها غير موجودة فعلاً أو غير مكتملة.

==================================================
سادساً: قواعد مهمة جداً
==================================================

1) لا تنشئ صفحة جديدة إذا كانت هناك صفحة حالية يمكن تطويرها
2) لا تنشئ API جديدة إذا كانت هناك API حالية يمكن تعديلها أو توسيعها
3) لا تنشئ جدول قاعدة بيانات جديد إذا كان الموجود يكفي
4) لا تكرر نفس الميزة بأسماء مختلفة
5) لا تغير منطق النظام القائم بدون حاجة
6) حافظ على هيكل النظام الحالي
7) أي إضافة جديدة يجب أن تكون مندمجة مع النظام الحالي
8) أي تطوير يجب أن يكون production-ready وليس شكلياً

==================================================
سابعاً: ما أريده في الرد النهائي منك
==================================================

بعد التنفيذ أريد منك تقريراً واضحاً ومنظماً يتضمن:

1) ما الذي كان موجوداً ويعمل
2) ما الذي كان موجوداً لكن تم إصلاحه
3) ما الذي كان موجوداً لكن تم تحسينه
4) ما الذي لم يكن موجوداً وتمت إضافته
5) ما الذي تم دمجه أو منع تكراره
6) ما الجداول التي استخدمتها
7) ما APIs التي استخدمتها أو عدلتها
8) ما الصفحات التي عدلتها
9) ما الذي تم اختباره فعلياً
10) أي ملاحظات مهمة عن النظام الحالي

==================================================
ثامناً: المطلوب العملي الآن
==================================================

ابدأ أولاً بمراجعة النظام الحالي مراجعة حقيقية وشاملة، ثم:
- أصلح الموجود
- أكمل الناقص
- أضف فقط غير الموجود
- وامنع التكرار تماماً

لا أريد بناء عشوائي أو إضافة مكررة.
أريد مراجعة ذكية + تحسين الموجود + إضافة غير الموجود فقط.

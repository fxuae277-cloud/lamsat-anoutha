أريد إعادة تصميم وتطوير صفحة "العملاء" داخل النظام الحالي بشكل احترافي وعملي، بحيث تصبح صفحة قوية ومناسبة لبرنامج POS / Retail احترافي، وتخدم المالك والإدارة بشكل واضح، مع تحسين واجهة العرض، تفاصيل العميل، كشف الحساب، والطباعة والإرسال.

المطلوب تنفيذه داخل المشروع الحالي مباشرة، وليس إنشاء صفحة منفصلة خارج النظام.

==================================================
أولاً: تطوير صفحة قائمة العملاء
==================================================

أريد تحسين صفحة "العملاء" الحالية لتصبح أكثر احترافية، وتشمل:

1) تحسين أعلى الصفحة
إضافة Header واضح يحتوي على:
- عنوان الصفحة: العملاء
- وصف صغير: إدارة العملاء، مشترياتهم، وفواتيرهم
- زر رئيسي: + إضافة عميل
- يكون التصميم مناسب للواجهة العربية RTL

2) إضافة مؤشرات سريعة أعلى الصفحة (Customer KPIs)
أريد بطاقات إحصائية أعلى الصفحة تعرض:
- إجمالي عدد العملاء
- العملاء الجدد هذا الشهر
- إجمالي مشتريات العملاء
- أفضل عميل حسب المشتريات
- عدد العملاء النشطين
- عدد العملاء غير النشطين

3) تحسين البحث والفلترة
أريد شريط أدوات أعلى الجدول يشمل:
- بحث بالاسم
- بحث برقم الهاتف
- فلتر العملاء النشطين / غير النشطين
- فلتر أفضل العملاء
- فلتر العملاء الجدد
- فلتر حسب الفرع
- فلتر حسب فترة التسجيل أو آخر زيارة

4) تحسين جدول العملاء
أريد الجدول يصبح أكثر اكتمالاً ويشمل الأعمدة التالية:
- الاسم
- الهاتف
- عدد الفواتير
- عدد الزيارات
- إجمالي المشتريات
- متوسط الفاتورة
- آخر زيارة
- الفرع الأكثر شراءً منه
- الحالة (نشط / غير نشط)
- الإجراءات

5) عمود الإجراءات
إضافة قائمة إجراءات لكل عميل تشمل:
- عرض التفاصيل
- تعديل
- حذف
- إنشاء فاتورة جديدة لهذا العميل
- عرض كشف الحساب
- طباعة كشف الحساب
- إرسال كشف الحساب

6) تحسين تصميم الجدول
- جعل الجدول منسقاً بشكل نظيف
- دعم الترتيب sorting على الأعمدة المهمة
- دعم pagination إذا كان عدد العملاء كبيراً
- تحسين العرض على الكمبيوتر
- تحسين العرض على الهاتف

==================================================
ثانياً: تطوير نافذة / صفحة تفاصيل العميل
==================================================

أريد عند الضغط على العميل أن تظهر نافذة أو صفحة تفاصيل احترافية، مقسمة بشكل واضح، وتحتوي على:

1) بيانات العميل الأساسية
- الاسم
- رقم الهاتف
- تاريخ إضافة العميل
- آخر زيارة
- عدد الزيارات
- عدد الفواتير
- إجمالي المشتريات
- متوسط قيمة الفاتورة
- الفرع الأكثر تعاملًا معه
- ملاحظات العميل إن وجدت
- حالة العميل (نشط / غير نشط)

2) أزرار سريعة داخل تفاصيل العميل
أريد أزرار واضحة داخل تفاصيل العميل:
- تعديل العميل
- إنشاء فاتورة جديدة
- كشف حساب
- طباعة كشف الحساب
- تحميل PDF
- إرسال عبر واتساب

3) قسم فواتير العميل
إنشاء جدول فواتير العميل داخل التفاصيل ويشمل:
- رقم الفاتورة
- التاريخ
- الفرع
- المبلغ
- طريقة الدفع
- الحالة
- زر عرض الفاتورة

4) قسم ملخص العميل
إضافة بطاقات أو قسم ملخص يحتوي على:
- إجمالي المشتريات
- عدد الفواتير
- متوسط الفاتورة
- آخر عملية شراء
- عدد المرتجعات إن وجدت
- إجمالي المرتجعات إن وجدت

5) قسم النشاط / السجل
أريد قسم "آخر نشاط للعميل" ويشمل:
- آخر فاتورة
- آخر زيارة
- آخر مرتجع
- آخر عملية تمت على حسابه

==================================================
ثالثاً: إضافة كشف حساب العميل بشكل احترافي
==================================================

أريد إضافة ميزة "كشف حساب العميل" داخل النظام، بحيث يمكن عرض كشف الحساب وطباعته وإرساله.

1) زر كشف الحساب
إضافة زر واضح باسم:
- كشف حساب

داخل:
- قائمة الإجراءات في الجدول
- وداخل تفاصيل العميل

2) نافذة أو صفحة كشف الحساب
عند فتح كشف الحساب، أريد شاشة تحتوي على:

أ) بيانات العميل:
- الاسم
- الهاتف
- الفرع
- عدد الفواتير
- إجمالي المشتريات

ب) فلترة فترة الكشف:
- من تاريخ
- إلى تاريخ
- أزرار سريعة:
  - هذا الشهر
  - آخر 30 يوم
  - آخر 3 أشهر
  - كل الفترات

ج) جدول كشف الحساب ويشمل:
- التاريخ
- رقم الفاتورة
- نوع العملية (بيع / مرتجع / دفعة إن وجدت)
- الفرع
- طريقة الدفع
- المبلغ
- الملاحظات

د) ملخص أسفل الكشف:
- إجمالي المبيعات
- إجمالي المرتجعات
- صافي التعامل
- عدد العمليات
- متوسط الفاتورة

==================================================
رابعاً: الطباعة و PDF والإرسال
==================================================

أريد أن يدعم كشف حساب العميل ما يلي:

1) الطباعة
- زر طباعة كشف الحساب
- تصميم طباعة نظيف واحترافي
- مناسب لطباعة A4
- يحتوي على:
  - شعار لمسة أنوثة
  - اسم المحل
  - بيانات العميل
  - جدول كشف الحساب
  - الإجماليات
  - تاريخ الطباعة

2) تحميل PDF
- زر تحميل PDF
- يتم إنشاء ملف PDF حقيقي ومرتب
- اسم الملف يكون واضحاً مثل:
  customer-statement-{customerName}-{date}.pdf

3) إرسال عبر واتساب
- إضافة زر "إرسال عبر واتساب"
- إذا كان رقم العميل موجوداً:
  - يتم تجهيز رابط واتساب
  - مع رسالة جاهزة
  - وإرفاق أو تجهيز كشف الحساب PDF إذا كانت البنية الحالية تسمح بذلك
- وإن لم يكن الإرفاق المباشر متاحاً حالياً:
  - افتح واتساب برسالة جاهزة مع إمكانية تنزيل PDF أولاً

4) نص الرسالة الجاهزة
أريد رسالة واتساب افتراضية مثل:
السلام عليكم،
هذا كشف حساب مشترياتكم من لمسة أنوثة للفترة المحددة.
شكراً لتعاملكم معنا.

==================================================
خامساً: تحسين إضافة / تعديل العميل
==================================================

أريد تحسين نموذج إضافة وتعديل العميل بحيث يشمل:
- الاسم
- الهاتف
- ملاحظات
- الفرع الافتراضي إن لزم
- حالة العميل
- التحقق من صحة رقم الهاتف
- منع التكرار إذا كان رقم الهاتف موجوداً مسبقاً
- رسائل خطأ واضحة
- رسائل نجاح واضحة

==================================================
سادساً: التكامل مع المبيعات والفواتير
==================================================

أريد ربط صفحة العملاء بشكل أقوى مع باقي النظام:

1) من داخل العميل:
- زر إنشاء فاتورة جديدة مباشرة لهذا العميل
- عند إنشاء فاتورة يتم تعبئة بيانات العميل تلقائياً

2) من داخل المبيعات:
- إذا تم اختيار عميل، يتم تحديث:
  - عدد زياراته
  - إجمالي مشترياته
  - آخر زيارة
  - عدد فواتيره
  بشكل تلقائي بعد حفظ الفاتورة

3) إذا كان هناك مرتجع:
- يتم احتسابه داخل بيانات العميل وكشف الحساب

==================================================
سابعاً: تحسينات واجهة وتجربة الاستخدام
==================================================

أريد الصفحة تكون احترافية جداً من ناحية UX/UI:

- تصميم RTL عربي كامل
- مسافات جيدة
- بطاقات واضحة
- جدول منظم
- ألوان متناسقة مع هوية لمسة أنوثة
- أزرار واضحة وسهلة
- نافذة التفاصيل منظمة
- دعم الهاتف والتابلت والكمبيوتر
- إذا كانت الصفحة على الهاتف:
  - يتحول الجدول إلى بطاقات أو عرض مناسب للهاتف

==================================================
ثامناً: APIs المطلوبة أو المطلوب تعديلها
==================================================

أريد إنشاء أو تعديل APIs اللازمة لدعم هذه المزايا، مثل:

- GET /api/customers
- GET /api/customers/:id
- POST /api/customers
- PATCH /api/customers/:id
- DELETE /api/customers/:id
- GET /api/customers/:id/invoices
- GET /api/customers/:id/statement
- GET /api/customers/:id/statement/pdf
- GET /api/customers/kpis
- GET /api/customers/filters/options

إذا كانت بعض APIs موجودة:
- عدّلها بدل تكرارها
- ونظّم الاستجابة لتخدم الصفحة الجديدة بشكل صحيح

==================================================
تاسعاً: الحماية والصلاحيات
==================================================

أريد تطبيق الصلاحيات بشكل صحيح:
- فقط من يملك صلاحية العملاء يمكنه الدخول للصفحة
- فقط من يملك صلاحية التعديل يمكنه تعديل العميل
- فقط من يملك صلاحية الحذف يمكنه حذف العميل
- فقط من يملك صلاحية طباعة/تقارير يمكنه طباعة أو تحميل كشف الحساب
- تسجيل كل العمليات المهمة في activity log

==================================================
عاشراً: الاختبار المطلوب
==================================================

بعد التنفيذ أريد اختباراً فعلياً يشمل:
- عرض قائمة العملاء
- البحث والفلترة
- فتح تفاصيل العميل
- عرض فواتيره
- إنشاء كشف حساب
- طباعة كشف الحساب
- تحميل PDF
- تجربة إرسال عبر واتساب
- إنشاء عميل جديد
- تعديل عميل
- منع تكرار رقم الهاتف
- التأكد أن الإحصائيات تتحدث بشكل صحيح

==================================================
الحادي عشر: المطلوب في الرد النهائي
==================================================

بعد الانتهاء أريد تقريراً واضحاً يتضمن:
- الملفات التي تم تعديلها
- الملفات أو المكونات الجديدة التي تمت إضافتها
- APIs التي تم إنشاؤها أو تعديلها
- ما الذي تم تحسينه في صفحة العملاء
- كيف أستخدم كشف الحساب والطباعة وPDF وواتساب
- أي ملاحظات تقنية مهمة

مهم جداً:
- أريد التنفيذ داخل النظام الحالي
- لا أريد مجرد تحسين شكلي فقط
- أريد Frontend + Backend + APIs + طباعة + PDF + واتساب
- اجعل الصفحة production-ready واحترافية فعلاً
- حسّن صفحة العملاء الحالية بدلاً من إنشاء شيء منفصل غير مرتبط بالنظام

أريدك أن تصلح ميزة النسخة الاحتياطية داخل مشروعي الحالي على Replit إصلاحاً كاملاً، بحيث تعمل من صفحة الإعدادات داخل النظام، وتنتج ملف Backup حقيقي قابل للتنزيل والاستعادة على خادم Hetzner بدون فقدان البيانات.

نفّذ المطلوب مباشرة داخل المشروع الحالي، ولا تنشئ حلاً منفصلاً.

المطلوب الوظيفي:
1) إصلاح زر "إنشاء نسخة احتياطية" ليقوم بإنشاء نسخة حقيقية وليس ملفاً فارغاً.
2) النسخة الاحتياطية يجب أن تشمل:
   - قاعدة البيانات كاملة
   - جميع الجداول والبيانات
   - الإعدادات
   - الفروع
   - المستخدمين والصلاحيات
   - المنتجات
   - المخزون
   - المبيعات
   - الطلبات
   - الشفتات
   - المصروفات
   - المشتريات
   - الموردين
   - أي صور أو ملفات مرفوعة داخل uploads أو أي مجلد مشابه
3) إخراج النسخة في ملف ZIP واحد منظم بهذا الشكل:
   - /database
   - /uploads
   - /config
   - metadata.json
   - README-RESTORE.txt
4) اسم الملف يكون واضحاً مثل:
   lamsat-backup-YYYY-MM-DD-HH-mm.zip
5) بعد الإنشاء تظهر النسخة في صفحة الإعدادات مع:
   - اسم الملف
   - التاريخ
   - الحجم
   - الحالة
   - زر تنزيل
   - زر حذف
   - زر فحص سلامة

المطلوب التقني:
6) إذا كانت قاعدة البيانات PostgreSQL:
   - أنشئ dump صحيح وكامل
   - تأكد أن الملف الناتج غير فارغ
   - استخدم طريقة مناسبة للاستعادة على Hetzner
7) إذا كانت SQLite:
   - انسخ ملف قاعدة البيانات الحقيقي بعد التحقق من سلامته
8) أضف أو أصلح API endpoints التالية:
   - POST /api/settings/backup/create
   - GET /api/settings/backups
   - GET /api/settings/backup/download/:filename
   - DELETE /api/settings/backup/:filename
   - POST /api/settings/backup/validate/:filename
9) طبّق صلاحيات صحيحة:
   - فقط المالك أو الأدمن يمكنه إنشاء/تنزيل/حذف النسخ
   - منع أي path traversal أو قراءة ملفات خارج مجلد backup
10) أضف logging واضح لعملية النسخ:
   - وقت البداية
   - وقت النهاية
   - اسم الملف
   - الحجم
   - نجاح/فشل تصدير قاعدة البيانات
   - نجاح/فشل ضغط الملفات
   - سبب الخطأ إن وجد

التحقق والجودة:
11) بعد إنشاء أي نسخة نفّذ validation تلقائي:
   - ملف ZIP موجود
   - حجمه أكبر من صفر
   - يحتوي database dump
   - يحتوي metadata.json
   - يحتوي README-RESTORE.txt
   - يحتوي uploads إن وجدت
12) لا أريد تنفيذ شكلي. أريد اختباراً فعلياً بعد الإصلاح:
   - إنشاء نسخة احتياطية حقيقية
   - تنزيلها
   - فحص محتوياتها
   - التأكد أنها ليست فارغة
   - عرض نتيجة الاختبار بوضوح

ملف README-RESTORE:
13) أنشئ داخل النسخة ملف README-RESTORE.txt يشرح:
   - كيف أنقل النسخة إلى Hetzner
   - كيف أفك الضغط
   - كيف أستعيد قاعدة البيانات
   - أين أضع uploads
   - ما متغيرات البيئة المطلوبة إن وجدت

مهم:
14) أصلح الميزة داخل النظام الحالي Frontend + Backend.
15) إذا كانت هناك مكتبات ناقصة للضغط أو التصدير أو إدارة الملفات فأضفها.
16) إذا كانت المشكلة في المسارات أو الصلاحيات أو طريقة التصدير أو التنزيل أو أن الملف فارغ، أصلحها بالكامل.
17) اجعل الحل production-ready وليس مؤقتاً.
18) بعد الانتهاء أعطني تقريراً واضحاً يتضمن:
   - الملفات التي تم تعديلها
   - سبب المشكلة الأصلية
   - ما الذي تم إصلاحه
   - كيف أستخدم النسخة الاحتياطية من داخل الإعدادات
   - كيف أرفعها وأستعيدها على Hetzner

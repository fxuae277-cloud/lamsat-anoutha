أريد إنشاء نسخة هاتف احترافية للموظف داخل النظام الحالي تعمل كتطبيق PWA على الهاتف (Android و iPhone) بدون نشر في المتاجر. هذه النسخة يجب أن تكون مخصصة للموظف فقط، ومختلفة عن نسخة المالك، وتحتوي فقط على الوظائف التشغيلية اليومية اللازمة للبيع والعمل داخل الفرع.

المطلوب بالتفصيل:

أولاً: صلاحيات الموظف
- المستخدم الذي role = employee يرى فقط نسخة الموظف
- لا يرى أي صفحات خاصة بالمالك
- لا يرى التقارير العامة
- لا يرى الإعدادات الحساسة
- لا يرى النسخ الاحتياطية
- لا يرى المستخدمين أو الصلاحيات
- لا يرى كل الفروع، فقط فرعه المحدد

ثانياً: الصفحة الرئيسية للموظف
أنشئ Home Mobile بسيطة جداً تحتوي على:
- اسم الموظف
- اسم الفرع
- حالة الشفت الحالية (مفتوح / مغلق)
- إجمالي مبيعاته اليوم
- عدد فواتيره اليوم
- زر فتح الشفت
- زر إغلاق الشفت
- زر بدء بيع جديد
- زر عرض الفواتير
- زر مسح باركود

ثالثاً: شاشة البيع POS على الهاتف
أريد شاشة بيع مخصصة للهاتف تشمل:
- مسح الباركود بالكاميرا
- إضافة المنتج مباشرة بعد قراءة الباركود
- إمكانية البحث عن المنتج بالاسم
- عرض المنتجات المضافة في الفاتورة
- تعديل الكمية
- حذف المنتج من الفاتورة
- عرض السعر
- حساب الإجمالي تلقائياً
- اختيار طريقة الدفع
- حفظ الفاتورة
- طباعة الفاتورة إذا كانت الطباعة متاحة
- منع إتمام البيع إذا لم يكن الشفت مفتوحاً

رابعاً: إدارة الشفت للموظف
إنشاء صفحة أو نافذة للشفت تشمل:
- فتح الشفت
- إدخال المبلغ الافتتاحي
- عرض وقت فتح الشفت
- عرض المبيعات أثناء الشفت
- إغلاق الشفت
- إدخال المبلغ الفعلي في الصندوق عند الإغلاق
- حساب الفرق النقدي تلقائياً
- منع الإغلاق إذا وجدت فواتير معلقة pending
- عرض نتيجة الإغلاق بوضوح

خامساً: صفحة الفواتير
إنشاء صفحة Mobile تعرض:
- فواتير اليوم الخاصة بالموظف
- رقم الفاتورة
- الوقت
- المبلغ
- حالة الفاتورة
- طريقة الدفع
- إمكانية فتح الفاتورة لعرض التفاصيل
- لا يسمح للموظف بحذف أو تعديل الفواتير إلا حسب الصلاحية الموجودة بالنظام

سادساً: صفحة مسح الباركود
أنشئ تجربة سهلة للهاتف:
- زر كبير لفتح الكاميرا
- بعد قراءة الباركود يتم:
  - جلب المنتج مباشرة
  - إضافته إلى الفاتورة أو عرضه
- إذا لم يوجد المنتج:
  - تظهر رسالة واضحة
- دعم الاستخدام السريع أثناء البيع

سابعاً: التنقل في نسخة الموظف
أريد شريط سفلي بسيط يحتوي على:
- الرئيسية
- البيع
- الفواتير
- الشفت
- المزيد

وفي "المزيد" فقط:
- بياناتي
- تسجيل الخروج

ثامناً: تحويل النظام إلى PWA
- دعم Add to Home Screen
- واجهة Mobile Responsive
- أيقونة للتطبيق
- Manifest + Service Worker
- العمل الجزئي عند ضعف الإنترنت بما لا يسبب أخطاء في البيع

تاسعاً: APIs المطلوبة
- GET /api/mobile/employee/home
- POST /api/mobile/shift/open
- POST /api/mobile/shift/close
- POST /api/mobile/sale/create
- GET /api/mobile/my-invoices
- GET /api/products/barcode/:code

عاشراً: القيود الأمنية
- الموظف يرى فرعه فقط
- الموظف يرى بياناته فقط
- منع الوصول لأي API تخص المالك
- حماية كل الصفحات والطلبات حسب role و branch_id
- تسجيل كل العمليات في activity log

الحادي عشر: تحسين واجهة الهاتف
- أزرار كبيرة
- خطوط واضحة
- خطوات قليلة
- سرعة في إدخال البيع
- عدم ازدحام الصفحة
- التركيز على البيع والشفت فقط

الثاني عشر: الاختبار
بعد التنفيذ اختبر فعلياً:
- فتح الشفت
- مسح باركود
- إنشاء فاتورة
- إغلاق الشفت
- منع الإغلاق عند وجود pending invoice
- ظهور البيانات الصحيحة للموظف فقط
- التأكد أن الموظف لا يستطيع رؤية صلاحيات المالك

المطلوب النهائي:
نسخة هاتف للموظف داخل النظام الحالي تعمل كتطبيق PWA، تكون خفيفة وسريعة ومخصصة للبيع والشفت والفواتير فقط، بدون أي صلاحيات إدارية.

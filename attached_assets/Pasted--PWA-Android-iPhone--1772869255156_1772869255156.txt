أريد تحويل النظام الحالي إلى نسخة هاتف احترافية تعمل كتطبيق PWA على الهاتف (Android و iPhone) بدون نشر في المتاجر، وتكون داخل نفس المشروع الحالي، وليس مشروعاً جديداً.

المطلوب أن يحتوي التطبيق على نسختين داخل نفس النظام:
1) نسخة خاصة بالمالك
2) نسخة خاصة بالموظف

ويتم تحديد ما يظهر لكل مستخدم حسب:
- role
- branch_id
- الصلاحيات الحالية بالنظام

==================================================
أولاً: المتطلبات العامة
==================================================

1) تحويل النظام إلى PWA
- إضافة manifest.json
- إضافة service-worker.js
- دعم Add to Home Screen
- إضافة أيقونة للتطبيق
- واجهة Mobile Responsive حقيقية
- تصميم مناسب للهاتف وليس مجرد تصغير نسخة الكمبيوتر
- دعم العمل الجزئي عند ضعف الإنترنت بدون كسر النظام

2) نفس رابط النظام الحالي
- لا أريد مشروعاً منفصلاً
- أريد التنفيذ داخل النظام الحالي
- عند فتح النظام من الهاتف يتم عرض نسخة الهاتف المناسبة حسب المستخدم

3) نظام صلاحيات واضح
- owner يرى نسخة المالك فقط
- employee يرى نسخة الموظف فقط
- منع الوصول لأي صفحات أو APIs غير مصرح بها
- حماية جميع الصفحات والطلبات حسب role و branch_id

4) التنقل في الهاتف
إنشاء شريط سفلي Mobile Bottom Navigation
ويختلف حسب نوع المستخدم

==================================================
ثانياً: نسخة المالك على الهاتف
==================================================

أريد نسخة هاتف احترافية للمالك تحتوي على 5 أقسام رئيسية في الشريط السفلي:
- الرئيسية
- المشتريات
- التحويل
- الجرد
- المزيد

----------------------------------
1) الصفحة الرئيسية للمالك (Dashboard)
----------------------------------
أنشئ لوحة تحكم Mobile خاصة بالمالك وتحتوي على:

المؤشرات الرئيسية:
- مبيعات اليوم
- مبيعات الشهر
- عدد الفواتير
- متوسط الفاتورة
- الربح التقريبي
- المصروفات
- النقد الحالي
- عدد الفروع المفتوحة الآن
- عدد الشفتات المفتوحة
- المخزون المنخفض
- المنتجات النافدة
- التحويلات المعلقة
- الجرد غير المعتمد
- أفضل 5 منتجات مبيعاً
- أفضل فرع اليوم
- آخر العمليات في النظام

داخل اللوحة أريد:
- فلتر: كل الفروع / فرع محدد
- فلتر: اليوم / 7 أيام / 30 يوم / هذا الشهر / فترة مخصصة
- رسم بياني للمبيعات
- جدول مختصر لأداء الفروع
- قسم تنبيهات مهم
- قسم آخر العمليات
- أزرار سريعة:
  - فاتورة مشتريات جديدة
  - تحويل جديد
  - جرد جديد
  - مسح باركود

----------------------------------
2) صفحة المشتريات للمالك
----------------------------------
إنشاء صفحة Mobile لإدخال فواتير المشتريات من الهاتف وتشمل:
- اختيار المورد
- إدخال رقم فاتورة المورد
- التاريخ
- ملاحظة
- رفع صورة الفاتورة
- زر مسح باركود المنتج بالكاميرا
- إضافة المنتج تلقائياً بعد قراءة الباركود
- إمكانية البحث عن المنتج
- إدخال الكمية
- إدخال سعر الشراء
- حساب إجمالي الصنف
- حساب إجمالي الفاتورة تلقائياً
- زر حفظ كمسودة
- زر اعتماد الفاتورة وتحويلها إلى المخزون

----------------------------------
3) صفحة التحويل بين المخازن والفروع
----------------------------------
إنشاء صفحة Mobile للتحويل تشمل:
- اختيار المصدر (فرع/مخزن)
- اختيار الوجهة (فرع/مخزن)
- مسح المنتجات بالباركود
- إضافة المنتجات يدوياً إذا لزم
- عرض المخزون المتوفر
- إدخال الكمية
- ملاحظة التحويل
- زر إرسال التحويل
- عرض الحالة:
  - مرسل
  - قيد الاستلام
  - مكتمل

----------------------------------
4) صفحة الجرد للمالك
----------------------------------
إنشاء صفحة Mobile للجرد تشمل:
- اختيار الفرع أو المخزن
- اختيار الفئة أو القسم إن وجد
- زر بدء جرد
- مسح المنتج بالباركود
- عرض اسم المنتج
- عرض الكمية المسجلة في النظام
- إدخال الكمية الفعلية
- حساب الفرق تلقائياً
- حفظ كل سطر جرد
- زر اعتماد الجرد
- زر اعتماد التسوية

----------------------------------
5) صفحة المزيد للمالك
----------------------------------
تحتوي على:
- التقارير
- الموردون
- المنتجات
- المخازن
- سجل النشاط
- الإعدادات
- النسخ الاحتياطية
- تسجيل الخروج

==================================================
ثالثاً: نسخة الموظف على الهاتف
==================================================

أريد نسخة هاتف خفيفة للموظف تحتوي على 5 أقسام في الشريط السفلي:
- الرئيسية
- البيع
- الفواتير
- الشفت
- المزيد

----------------------------------
1) الصفحة الرئيسية للموظف
----------------------------------
تحتوي على:
- اسم الموظف
- اسم الفرع
- حالة الشفت الحالية (مفتوح / مغلق)
- إجمالي مبيعاته اليوم
- عدد فواتيره اليوم
- زر فتح الشفت
- زر إغلاق الشفت
- زر بدء بيع جديد
- زر عرض الفواتير
- زر مسح باركود

----------------------------------
2) شاشة البيع POS للموظف
----------------------------------
إنشاء شاشة بيع مخصصة للهاتف تشمل:
- مسح الباركود بالكاميرا
- إضافة المنتج مباشرة بعد قراءة الباركود
- إمكانية البحث بالاسم
- عرض المنتجات المضافة في الفاتورة
- تعديل الكمية
- حذف المنتج
- عرض السعر
- حساب الإجمالي تلقائياً
- اختيار طريقة الدفع
- حفظ الفاتورة
- طباعة الفاتورة إذا كانت الطباعة متاحة
- منع إتمام البيع إذا لم يكن الشفت مفتوحاً

----------------------------------
3) صفحة الفواتير للموظف
----------------------------------
تعرض:
- فواتير اليوم الخاصة بالموظف
- رقم الفاتورة
- الوقت
- المبلغ
- حالة الفاتورة
- طريقة الدفع
- عرض التفاصيل
- لا يسمح بالحذف أو التعديل إلا حسب الصلاحية الموجودة

----------------------------------
4) صفحة الشفت للموظف
----------------------------------
تشمل:
- فتح الشفت
- إدخال المبلغ الافتتاحي
- عرض وقت فتح الشفت
- عرض مبيعات الشفت
- إغلاق الشفت
- إدخال المبلغ الفعلي عند الإغلاق
- حساب الفرق النقدي تلقائياً
- منع الإغلاق إذا وجدت فواتير pending
- عرض نتيجة الإغلاق بوضوح

----------------------------------
5) صفحة المزيد للموظف
----------------------------------
تحتوي فقط على:
- بياناتي
- تسجيل الخروج

==================================================
رابعاً: الباركود والكاميرا
==================================================

أريد دعم كامل للباركود عبر كاميرا الهاتف في:
- المشتريات
- البيع
- التحويل
- الجرد

المطلوب:
- زر كبير وواضح لفتح الكاميرا
- بعد قراءة الباركود:
  - جلب المنتج مباشرة
  - إضافته للصفحة الحالية
- إذا لم يوجد المنتج:
  - تظهر رسالة واضحة
- جعل التجربة سريعة ومناسبة للهاتف

==================================================
خامساً: APIs المطلوبة
==================================================

أنشئ أو عدّل APIs التالية حسب الحاجة:

للـ Dashboard:
- GET /api/mobile/owner/dashboard

للمشتريات:
- POST /api/purchases/mobile-create

للتحويل:
- POST /api/transfers/mobile-create

للجرد:
- POST /api/inventory/mobile-count

للبيع:
- POST /api/mobile/sale/create

للشفت:
- POST /api/mobile/shift/open
- POST /api/mobile/shift/close

للفواتير:
- GET /api/mobile/my-invoices

للباركود:
- GET /api/products/barcode/:code

==================================================
سادساً: الحماية والأمان
==================================================

1) المالك فقط يرى صفحات المالك وواجهاته وAPIs الخاصة به
2) الموظف فقط يرى صفحات الموظف وواجهاته وAPIs الخاصة به
3) الموظف لا يرى:
- لوحة تحكم المالك
- المشتريات
- التحويلات
- الجرد
- التقارير العامة
- الإعدادات الحساسة
- النسخ الاحتياطية
- المستخدمين والصلاحيات
4) كل مستخدم يرى بيانات فرعه فقط إلا المالك
5) حماية كل الطلبات حسب role و branch_id
6) تسجيل كل العمليات في activity log

- خطوات قصيرة
- شاشات واضحة
- عدم ازدحام
- سرعة في الاستخدام
- خطوط مناسبة للهاتف
- بطاقات واضحة للمؤشرات
- تصميم احترافي وسريع

==================================================
ثامناً: الاختبار المطلوب
==================================================

بعد التنفيذ أريد اختباراً فعلياً يشمل:

اختبار نسخة المالك:
- فتح Dashboard
- فلترة المؤشرات
- مسح باركود في المشتريات
- إنشاء فاتورة مشتريات
- إنشاء تحويل
- تنفيذ جرد
- التأكد أن النتائج تظهر فوراً عند توفر الإنترنت

اختبار نسخة الموظف:
- فتح الشفت
- مسح باركود
- إنشاء فاتورة بيع
- عرض الفواتير
- إغلاق الشفت
- منع الإغلاق إذا فيه pending invoice
- التأكد أن الموظف لا يرى صفحات المالك

==================================================
تاسعاً: المطلوب في الرد النهائي
==================================================

بعد الانتهاء أريد تقريراً واضحاً يتضمن:
- الملفات التي تم تعديلها
- الصفحات التي تمت إضافتها
- APIs التي تم إنشاؤها أو تعديلها
- كيف يتم التمييز بين نسخة المالك ونسخة الموظف
- كيف أثبت التطبيق على الهاتف كـ PWA
- ما الذي تم اختباره فعلياً
- أي ملاحظات تقنية مهمة

مهم جداً:
- لا تنشئ مشروعاً جديداً
- نفّذ كل شيء داخل النظام الحالي
- اجعل التطبيق عملياً وسريعاً للهاتف
- أريد Frontend + Backend + حماية + Barcode + PWA
- الحل يجب أن يكون production-ready وليس شكلياً

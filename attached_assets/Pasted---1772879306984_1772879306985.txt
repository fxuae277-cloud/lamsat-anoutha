أريد تطوير صفحة وكشف الرواتب الحالي داخل النظام الحالي لتصبح صفحة "كشف رواتب احترافي للشركة" بشكل عملي ومحاسبي واضح، مع الحفاظ على الموجود وعدم تكرار أي ميزة موجودة بالفعل.

مهم جداً:
- لا أريد إنشاء شيء مكرر إذا كانت بعض العناصر موجودة
- راجع صفحة الرواتب الحالية أولاً
- طوّر الموجود
- أصلح الناقص
- أضف فقط ما هو غير موجود فعلاً
- نفّذ داخل المشروع الحالي فقط

==================================================
أولاً: مراجعة صفحة الرواتب الحالية قبل التعديل
==================================================

أريد أولاً مراجعة صفحة الرواتب الحالية وكشف الراتب الموجود حالياً، والتأكد من:
1) ما الأعمدة الموجودة الآن
2) ما العمليات الموجودة فعلياً
3) هل يوجد ربط حقيقي بين:
   - الموظف
   - الفرع
   - الراتب الأساسي
   - العمولات
   - الخصومات
   - السلف
   - صافي الراتب
4) هل يوجد احتساب صحيح فعلاً أم مجرد عرض
5) هل يوجد نظام دفع أو فقط عرض كشف
6) هل يوجد تصدير أو طباعة أو لا
7) هل يوجد حفظ لحالة الدفع أو لا

إذا كانت بعض الميزات موجودة:
- لا تكررها
- فقط حسّنها أو أكملها

==================================================
ثانياً: تطوير كشف الرواتب الحالي
==================================================

أريد أن يصبح كشف الرواتب الشهري أو نافذة "تفاصيل كشف الراتب - مارس 2026" احترافية وواضحة وتشمل الأعمدة التالية:

- الموظف
- الفرع
- الراتب الأساسي
- العمولات
- إجمالي الراتب قبل الخصومات
- الخصومات
- السلف
- صافي الراتب
- حالة الدفع
- الإجراءات

إذا كان بعض هذه الأعمدة موجوداً بالفعل:
- أبقه
- حسّن ترتيبه وعرضه
- لا تنشئ تكراراً

==================================================
ثالثاً: طريقة الحساب المطلوبة
==================================================

أريد التأكد أن الحساب يكون واضحاً وصحيحاً كالتالي:

إجمالي الراتب قبل الخصومات = الراتب الأساسي + العمولات
صافي الراتب = إجمالي الراتب قبل الخصومات - الخصومات - السلف

ويجب عرض هذه القيم بوضوح لكل موظف.

إذا كان النظام الحالي يحسبها بالفعل:
- راجع وتأكد من صحتها
- أصلح أي خطأ
- لا تعيد البناء بدون حاجة

==================================================
رابعاً: إضافة حالة الدفع
==================================================

أريد إضافة حالة دفع لكل موظف داخل كشف الرواتب، مثل:

- غير مدفوع
- مدفوع
- مدفوع جزئيًا

وإذا كانت هناك حالة موجودة حالياً:
- استخدمها
- أكملها
- لا تكررها

ويجب أن تظهر الحالة داخل الجدول بشكل واضح وملون.

==================================================

- تسجيل دفع الراتب
- تعديل الاستحقاقات أو الخصومات إذا كان النظام يسمح
- طباعة قسيمة الراتب
- تحميل PDF

إذا كانت بعض هذه الإجراءات موجودة:
- حسّنها

- لا تكررها

- تاريخ الدفع
- طريقة الدفع
- الفرع
- المستخدم الذي نفذ العملية
- ملاحظة إن وجدت

ويجب مراجعة هل توجد بنية حالية يمكن استخدامها:
- إذا يوجد جدول أو API أو سجل مناسب، استخدمه
- لا تنشئ تكراراً
- إذا غير موجود، أضفه بشكل صحيح

الثاني عشر: الربط مع النظام المالي

أريد مراجعة وربط الرواتب مع الجوانب المالية الحالية في النظام:

1) إذا تم دفع راتب:
- يجب أن ينعكس على المصروفات أو المحاسبة اليومية إذا كانت بنية النظام الحالية تدعم ذلك

2) إذا توجد صفحة مصروفات:
- لا تكرر العملية بشكل منفصل
- اربطها منطقياً أو سجّلها بشكل صحيح

3) إذا توجد سجلات مالية حالية:
- استعملها
- لا تنشئ منطقاً موازياً يسبب تضارباً

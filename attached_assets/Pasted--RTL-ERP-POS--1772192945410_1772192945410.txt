صمّم واجهة عربية بالكامل (RTL) لنظام ERP + POS لمحل “لمسة أنوثة إكسسوارات لوى” في عمان.

الهدف: نظام لإدارة 3 فروع + مخزن رئيسي + مخازن الفروع + كاشير + طلبات واتساب/انستجرام + مصروفات + رواتب + تقارير مالية.
العملة: ريال عماني (OMR) — ضريبة قيمة مضافة VAT 5% — الأسعار موحدة في كل الفروع.

الفروع (3):
1) لمسة أنوثة إكسسوارات لوى
2) لمسة أنوثة شناص
3) لمسة أنوثة (الفرع الثالث)

المستخدمون والصلاحيات:
- ملاك (2): كل الصلاحيات
- كاشير (3): POS + إغلاق وردية + إرجاع/خصم حسب الصلاحية
- موظف (3): مخزون + طلبات + استلام/تحويل مخزون (بدون محاسبة)

متطلبات التصميم (UI/UX):
- اللغة العربية فقط افتراضيًا + اتجاه RTL
- خط عربي واضح (يفضّل Tajawal أو Cairo) 
- ألوان ناعمة أنثوية (وردي/روز + رمادي فاتح) بدون مبالغة
- واجهة تشبه “دفترة” من حيث التنظيم (قائمة جانبية + لوحة تحكم + جداول + فلاتر)
- تصميم متجاوب: كمبيوتر + تابلت (للكاشير)

الأقسام (Pages) المطلوبة في القائمة الجانبية (كلها بالعربي):
1) لوحة التحكم
   - بطاقات: مبيعات اليوم (OMR) / إجمالي الضريبة اليوم / عدد الطلبات / تنبيهات نقص المخزون
   - رسوم مبسطة: مبيعات آخر 7 أيام
2) نقطة البيع (POS)
   - اختيار الفرع + اختيار الكاشير
   - بحث بالباركود/الاسم
   - سلة مشتريات
   - خصم (نسبة/قيمة) + ضريبة 5% تلقائيًا
   - طرق الدفع: نقدًا / تحويل بنكي
   - عند “تحويل بنكي”: رفع صورة إيصال + إدخال رقم العملية Txn ID + الوقت + المبلغ (يدوي الآن)
   - طباعة/إرسال فاتورة
   - إغلاق وردية + تقرير وردية
3) المنتجات والأسعار
   - إضافة/تعديل منتج
   - باركود، اسم، فئة، سعر موحد، صورة
   - حالة: نشط/غير نشط
4) المخزون
   - مخزن رئيسي + مخازن الفروع
   - استلام بضاعة للمخزن الرئيسي (بالمورّد + فاتورة شراء)
   - تحويل مخزون: رئيسي → فرع (مع باركود وكميات)
   - جرد + تنبيهات نقص المخزون
5) الطلبات (واتساب/انستجرام)
   - إدخال الطلب يدويًا (اسم العميل، جوال، مدينة، العنوان)
   - النظام يحدد أقرب فرع تلقائيًا حسب المدينة (قائمة مدن + ربطها بالفرع)
   - حالات الطلب: جديد / تم التجهيز / جاهز للاستلام / خرج للتوصيل / مكتمل / ملغي
   - خيار: استلام من الفرع / توصيل (توصيل خارجي)
6) العملاء
   - سجل العميل + طلباته + فواتيره + رصيده
7) الموردون والمشتريات
   - موردين + فواتير شراء + استلام مخزون
8) المصروفات (بالصورة)
   - زر “إضافة مصروف”
   - رفع صورة الفاتورة
   - إدخال: التاريخ، الفرع، المبلغ، التصنيف (إيجار/كهرباء/ماء/هواتف/رواتب/نثريات/منظفات/غيره)
   - ملاحظات
9) الرواتب والموظفين
   - بيانات الموظف + الفرع
   - راتب شهري + خصومات + سلف
   - تقرير رواتب شهري
10) المالية والتقارير
   - تقرير مبيعات حسب الفرع/الكاشير/المنتج
   - تقرير VAT 5%
   - ملخص أرباح وخسائر مبسط
   - تقرير المصروفات حسب التصنيف والفرع
11) الإعدادات
   - إدارة الفروع والمدن المرتبطة بها
   - إدارة المخازن
   - إدارة المستخدمين والصلاحيات
   - إعدادات الضريبة (5%) + العملة (OMR)

عناصر مهمة في التصميم:
- كل الجداول بالعربي: (الرقم، الاسم، الفرع، التاريخ، المبلغ، الحالة…)
- أزرار عربية واضحة: (إضافة، حفظ، تعديل، حذف، بحث، تصدير)
- دعم ماسح الباركود في POS والمخزون (حقل “امسح الباركود واضغط Enter” بالعربي)
- رسائل نجاح/خطأ عربية

المخرجات المطلوبة:
- Prototype كامل للتنقل بين الصفحات (Routing)
- مكونات جاهزة للشاشات الأساسية: Dashboard / POS / Inventory / Orders / Expenses / Settings
- واجهات إدخال بيانات (Dialogs/Forms) عربية لكل قسم

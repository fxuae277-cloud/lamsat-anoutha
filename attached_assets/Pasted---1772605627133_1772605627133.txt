أمر للمساعد — إلزام رقم معاملة البطاقة + خيار إرسال واتساب/طباعة

نفّذ التعديلات التالية في نافذة إتمام الدفع داخل POS:

1) واجهة الدفع (UI)

عند فتح نافذة “إتمام الدفع”:

أ) إضافة خيار طريقة استلام الفاتورة

أضف قسم باسم استلام الفاتورة يحتوي خيارات واضحة:

☑ إرسال عبر واتساب (يظهر ويعمل إذا تم إدخال رقم هاتف العميل)

☑ طباعة الفاتورة (يعمل دائماً إذا الطابعة متاحة)

اجعل المستخدم يستطيع اختيار:

واتساب فقط

طباعة فقط

الاثنين معاً

ب) رقم هاتف العميل

حقل: رقم هاتف العميل (اختياري)

إذا تم تفعيل “إرسال عبر واتساب” ولم يوجد رقم → اعرض خطأ:
"أدخل رقم هاتف العميل لإرسال الفاتورة عبر واتساب"

ج) طريقة الدفع

الأزرار: نقداً / بطاقة / تحويل بنكي

2) إلزام رقم معاملة البطاقة عند اختيار (بطاقة)

عند اختيار طريقة الدفع = بطاقة:

أظهر حقل إلزامي باسم:
رقم معاملة البطاقة (إلزامي)
(مثال: RRN / Auth / Reference)

قواعد التحقق (Validation):

لا يسمح بتأكيد الدفع إذا الحقل فارغ

يقبل أرقام فقط (0-9)

طول بين 6 و 20 رقم (قابل للتعديل حسب جهاز البنك لديك)

رسالة الخطأ:
"رقم معاملة البطاقة إلزامي لإتمام الدفع بالبطاقة"

عند اختيار الدفع ≠ بطاقة:

أخفِ الحقل

لا تطلبه

خزّنه NULL

3) قاعدة البيانات (DB)
أ) إضافة حقول للفاتورة/الدفع

أضف الحقول التالية (في جدول invoices أو sales أو payments حسب هيكل النظام الحالي):

payment_method (cash/card/bank)

card_txn_ref (text nullable) ✅ إلزامي فقط إذا card

customer_phone (text nullable)

receipt_channel (json/text) لتخزين خيارات الاستلام مثل: ["whatsapp"] أو ["print"] أو ["whatsapp","print"]

ملاحظات أمنية:

ممنوع حفظ رقم البطاقة الكامل أو CVV

لا تحفظ إلا رقم المعاملة المرجعي.

4) منطق تنفيذ “تأكيد الدفع”

عند الضغط على زر تأكيد الدفع:

التحقق قبل الإتمام:

إذا payment_method = card → card_txn_ref إلزامي

إذا خيار “واتساب” مفعّل → customer_phone إلزامي

بعد نجاح الدفع وإنشاء الفاتورة:

نفّذ حسب خيارات العميل:

إذا تم اختيار واتساب:

أنشئ نسخة رقمية للفاتورة (PDF أو رابط invoice_url)

أرسلها تلقائيًا عبر WhatsApp Business API/Cloud API

سجّل نتيجة الإرسال (success/failed) في logs

إذا تم اختيار طباعة:

اطبع الفاتورة فوراً

إذا تم اختيار الاثنين:

نفّذ الاثنين

5) التقارير والتفاصيل

أضف عرض “رقم معاملة البطاقة” في:

شاشة تفاصيل الفاتورة (عندما تكون طريقة الدفع بطاقة)

تقرير مبيعات البطاقة

تقرير التسوية اليومية (اختياري إن كان موجود)

6) تجربة المستخدم (UX) — مهم

الحقول الإلزامية تظهر بوضوح مع علامة (*)

لا تُبطئ الكاشير:

رقم المعاملة يظهر فقط عند “بطاقة”

رقم العميل اختياري إلا عند اختيار “واتساب”

النتيجة المطلوبة (Acceptance Criteria)

اختيار “بطاقة” بدون رقم معاملة → لا يسمح بتأكيد الدفع

اختيار “واتساب” بدون رقم عميل → لا يسمح بتأكيد الدفع

بعد الدفع تُرسل الفاتورة واتساب أو تُطبع حسب اختيار العميل

يتم حفظ رقم معاملة البطاقة داخل الفاتورة لسهولة المطابقة مع جهاز البنك

ما زالت مشكلة الأرقام العربية موجودة داخل حقول الإدخال الرقمية في بعض الصفحات، خصوصاً الحقول التي تستخدم input type="number" أو مكونات رقمية فيها أسهم زيادة/نقصان مثل:
- الكمية
- التكلفة الإضافية
- الشحن
- الجمارك
- التخليص
- أخرى

المشكلة الحالية:
حتى بعد تعديل التنسيق العام، ما زالت بعض الحقول تعرض الأرقام بالعربي مثل:
٠٠٢
٠٠٤
٠١
وهذا غير مقبول.
أريد إصلاحاً نهائياً لهذه الحقول نفسها، وليس فقط تنسيق العرض في الجداول.

المطلوب تنفيذه الآن بشكل مباشر:

1) ابحث في كامل المشروع عن كل الحقول الرقمية، خاصة:
- input type="number"
- أي NumericInput
- أي مكونات فيها stepper / arrows
- أي controlled inputs للأرقام

2) غيّر منطق الحقول الرقمية بحيث:
- تعرض الأرقام الإنجليزية فقط 0-9
- تحفظ الأرقام الإنجليزية فقط
- إذا أدخل المستخدم أرقاماً عربية مثل ١٢٣ أو ٠٠٤ يتم تحويلها فوراً إلى 123 و 004
- لا يسمح ببقاء Arabic-Indic digits في القيمة المعروضة

3) إذا كان input type="number" هو سبب المشكلة، فاستبدله في الأماكن المتأثرة بـ:
- input type="text"
مع inputMode="decimal" أو inputMode="numeric"
ومع onChange يقوم بـ:
- normalizeDigitsToEn
- السماح فقط بالأرقام الإنجليزية والنقطة العشرية عند الحاجة
- منع أي محارف رقمية عربية من البقاء في العرض

4) أنشئ مكوّن موحد على مستوى المشروع مثل:
EnglishNumberInput
أو
NumericEnInput

ويكون مسؤولاً عن:
- تحويل الأرقام العربية إلى إنجليزية فوراً
- الحفاظ على القيمة كنص/رقم إنجليزي
- دعم الكسور العشرية
- دعم القيم المالية
- دعم الكميات
- دعم step/min/max إن لزم
- عدم إظهار الأرقام بالعربية نهائياً

5) استبدل به جميع الحقول الرقمية في الصفحات التالية على الأقل:
- المشتريات
- تفاصيل فاتورة المورد
- المنتجات
- التكلفة الإضافية
- الشحن
- الجمارك
- التخليص
- أخرى
- الكمية
- سعر التكلفة
- سعر البيع
- الخصومات
- السلف
- الرواتب
- أي modal فيه مبلغ أو كمية

6) طبّق CSS fix على هذه الحقول نفسها:
- direction: ltr للحقل الرقمي نفسه إذا لزم
- text-align مناسب
- unicode-bidi مناسب إن احتاج
- font-family يدعم Latin digits بوضوح
لكن بدون تخريب RTL العام للصفحة

7) تأكد أن placeholder والقيمة المعروضة داخل الحقل كلاهما بالأرقام الإنجليزية فقط.

8) افحص مصدر المشكلة أيضاً من أي formatter أو state initialization:
إذا كانت القيمة الأولية تأتي بالعربي من backend أو default values أو form reset أو zod parsing أو query data، قم بتحويلها فوراً إلى الإنجليزية قبل وضعها في input state.

9) أضف دالة مركزية مثل:
normalizeDigitsToEn(value)
تتعامل مع:
٠١٢٣٤٥٦٧٨٩
۰۱۲۳۴۵۶۷۸۹
وتحولها إلى:
0123456789

10) أضف دالة sanitizeNumericInput(value, { allowDecimal: true/false })
حتى تكون كل الحقول الرقمية تستخدم نفس السلوك.

11) بعد التنفيذ اختبر عملياً هذه الحالات:
- إدخال ١٢٣ => تصبح 123
- إدخال ٠٠٤ => تصبح 004
- إدخال ٢.٥ => تصبح 2.5
- عرض قيمة مخزنة ٠٠٢ => تظهر 002
- حقول stepper لا تعود تعرض أرقاماً عربية

12) لا أريد تغييراً شكلياً فقط.
أريد إصلاح مكوّنات الإدخال نفسها من الجذر، لأن المشكلة ما زالت موجودة في الحقول الرقمية تحديداً.

13) بعد التنفيذ أعطني ملخصاً محدداً:
- ما هو سبب المشكلة بالضبط
- هل تم استبدال input type="number" في الأماكن المتأثرة
- ما اسم المكوّن أو helper الجديد
- ما الصفحات التي تم إصلاحها فعلياً

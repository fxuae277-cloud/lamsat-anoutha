قراءة فواتير المورد بدقة أعلى (حتى مع صور غير واضحة)
1) رفع الصورة

في صفحة المشتريات → فاتورة شراء جديدة أضف زر:

رفع صورة الفاتورة (JPG / PNG / PDF)

بعد الرفع خزّن الملف مؤقتاً.

2) تحسين الصورة قبل OCR (Pre-processing)

طبّق المعالجات التالية لرفع دقة القراءة:

Deskew: تصحيح ميلان الصورة.

Grayscale: تحويلها لأبيض وأسود.

Adaptive Threshold: زيادة التباين.

Noise Removal: إزالة التشويش.

Crop: قص الحواف الفارغة.

يمكن استخدام: opencv (Python/Node bindings) أو أي مكتبة مماثلة.

3) تشغيل OCR

استخدم Tesseract OCR مع إعدادات مناسبة للجداول:

Language: eng

Page Segmentation Mode: --psm 6 (نص منظم)

Whitelist للأعمدة الرقمية: 0123456789.-

استخرج النص الكامل.

4) اكتشاف جدول الأصناف

بدلاً من قراءة النص كسطور عادية، ابحث عن نمط الأعمدة:

الأعمدة المتوقعة في الفواتير المشابهة:

NO | DESCRIPTION | QTY | PCS | PRICE | AMOUNT

استخدم Regex لاستخراج السطور التي تحتوي:

رقم تسلسلي

وصف

كمية

سعر

إجمالي

مثال Regex تقريبي:

(\d+)\s+([A-Z0-9\- ]+)\s+(\d+)\s+.*?\s+(\d+)\s+(\d+)

حوّل كل سطر إلى كائن:

{
  description,
  qty,
  price,
  amount
}
5) تحليل الوصف إلى (كود / لون / مقاس)

الفواتير لديك تحتوي وصف مثل:

K5126 MAROON 112+12

قسّم الوصف:

product_code = أول كلمة (K5126)

color = الكلمة الثانية (MAROON / BLACK / COFFEE…)

size = آخر جزء يحتوي أرقام (+ أو -)

Pseudo:

parts = description.split(" ")
product_code = parts[0]
color = parts[1]
size = parts[-1]
6) مطابقة الصنف مع النظام

لكل عنصر مستخرج:

ابحث في product_variants:

SELECT * FROM product_variants
WHERE sku = product_code
AND color = color
AND size = size

إذا وُجد → استخدم variant_id.

إذا لم يوجد → إنشاء Variant تلقائياً:

product_name = product_code

color

size

barcode = توليد داخلي (إن لم يوجد)

price = يمكن اقتراحه = price * معامل ربح افتراضي (اختياري)

7) إنشاء فاتورة شراء مسودة (Draft)

بعد تحليل كل العناصر:

أنشئ:

purchase_invoice
status = draft
supplier = المستخدم يختاره أو يستخرج من النص
invoice_number
date
location = WH-HQ

ثم أضف purchase_invoice_lines لكل عنصر:

variant_id
qty
unit_cost = price
line_total = amount
8) شاشة مراجعة قبل الحفظ

اعرض للمستخدم:

رقم الفاتورة

التاريخ

المورد

جدول الأصناف (قابل للتعديل)

الإجمالي

أزرار:

تعديل

حفظ كمسودة

اعتماد

9) الاعتماد (Post)

عند الضغط اعتماد:

زيادة المخزون في WH-HQ

إنشاء قيود في inventory_ledger

منع التعديل لاحقاً (Read-only)

10) تحسين الدقة بمرور الوقت (Learning)

احفظ نمط الفاتورة (Template) بناءً على:

موقع الجدول

ترتيب الأعمدة

عند رفع فواتير لاحقة من نفس المورد:

استخدم القالب لرفع الدقة إلى ~90%+.

معايير القبول (Acceptance Criteria)

رفع صورة فاتورة يولّد Purchase Invoice Draft خلال ثوانٍ.

≥80% من السطور تُستخرج تلقائياً.

أي باركود/Variant غير موجود يُنشأ عبر Quick Create.

لا يدخل المخزون إلا بعد اعتماد الفاتورة.

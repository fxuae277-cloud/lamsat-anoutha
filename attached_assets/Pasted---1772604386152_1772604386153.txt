أمر للمساعد — إضافة نظام العملاء وإرسال الفاتورة واتساب

قم بإضافة نظام إدارة العملاء وربطهم بالفواتير وإرسال الفاتورة عبر واتساب تلقائياً داخل نظام POS.

يجب تنفيذ الخصائص التالية:

1️⃣ إضافة جدول العملاء

إنشاء جدول جديد باسم:

customers

الحقول:

id (primary key)
name (text nullable)
phone (text unique)
created_at (timestamp)
total_spent (decimal default 0)
visits (integer default 0)
last_visit (timestamp)

ملاحظات:

رقم الهاتف يجب أن يكون unique

العميل يمكن إنشاؤه تلقائياً عند أول عملية شراء.

2️⃣ تعديل جدول الفواتير

إضافة حقل جديد في جدول الفواتير:

customer_id (integer nullable)

ويرتبط بـ:

customers.id
3️⃣ واجهة إدخال رقم العميل قبل الدفع

عند الضغط على زر:

دفع وإنهاء

يجب فتح نافذة منبثقة تحتوي على:

إدخال رقم الهاتف
[____________]

☑ إرسال الفاتورة عبر واتساب
☑ حفظ العميل

ملاحظات:

إدخال الرقم اختياري

إذا لم يتم إدخال رقم يتم إكمال الدفع بدون عميل.

4️⃣ منطق إنشاء العميل تلقائياً

عند إدخال رقم الهاتف:

النظام يجب أن:

يبحث في جدول customers

إذا وجد الرقم → يستخدم نفس العميل

إذا لم يوجد → ينشئ عميل جديد

مثال:

SELECT id FROM customers WHERE phone = ?

إذا لم يوجد:

INSERT INTO customers (phone, created_at)
5️⃣ تحديث بيانات العميل بعد كل عملية شراء

بعد إنشاء الفاتورة:

يجب تحديث:

visits +1
total_spent + invoice_total
last_visit = now()
6️⃣ ربط الفاتورة بالعميل

عند إنشاء الفاتورة يجب حفظ:

customer_id

داخل جدول الفواتير.

7️⃣ إنشاء صفحة العملاء داخل النظام

إنشاء صفحة جديدة في لوحة الإدارة باسم:

العملاء

تعرض جدول:

| العميل | الهاتف | عدد الزيارات | مجموع المشتريات | آخر زيارة |

البيانات من جدول:

customers
8️⃣ صفحة تفاصيل العميل

عند الضغط على أي عميل يجب عرض:

اسم العميل
رقم الهاتف
عدد الزيارات
إجمالي المشتريات

وقائمة بكل الفواتير المرتبطة به:

invoice id
date
total
branch
9️⃣ إرسال الفاتورة عبر واتساب تلقائياً

إذا كان خيار:

إرسال الفاتورة عبر واتساب

مفعل.

بعد إتمام الدفع يقوم النظام بإرسال رسالة واتساب تلقائياً.

استخدم:

WhatsApp Business API

أو

Meta WhatsApp Cloud API

الرسالة تكون:

مرحبا 🌸

شكراً لتسوقك من لمسة أنوثة

فاتورتك:

{items}

الإجمالي:
{total} ر.ع

الفرع:
{branch}

نتمنى زيارتك مرة أخرى
🔟 إنشاء نسخة رقمية من الفاتورة

يجب توليد:

Invoice PDF

أو رابط:

invoice_url

وإرساله داخل رسالة واتساب.

1️⃣1️⃣ تعديل واجهة POS

إضافة حقل صغير أعلى زر الدفع:

رقم العميل

أو يظهر في نافذة الدفع.

1️⃣2️⃣ حفظ الأرقام لأغراض التسويق

يجب أن تسمح صفحة العملاء لاحقاً:

إرسال عروض

معرفة العملاء الأكثر شراءً

استخراج قاعدة العملاء.

1️⃣3️⃣ عدم تعطيل سرعة الكاشير

مهم جداً:

إدخال الرقم اختياري

الدفع يجب أن يكون سريع

لا يتم إجبار الموظف على إدخال الرقم.

1️⃣4️⃣ الحماية

يجب:

منع تكرار نفس رقم الهاتف

تنظيف الرقم تلقائياً

مثال:

+96891234567
91234567

يتم حفظه بشكل موحد.

1️⃣5️⃣ الأداء

يجب أن تكون العمليات:

سريعة

غير مؤثرة على سرعة POS

تعمل في الخلفية (async).

النتيجة المطلوبة

بعد تنفيذ هذا النظام يجب أن يكون:

✔ النظام يجمع أرقام العملاء تلقائياً
✔ كل فاتورة مرتبطة بعميل
✔ إرسال الفاتورة واتساب مباشرة
✔ قاعدة بيانات عملاء للتسويق
✔ صفحة إدارة العملاء كاملة.

# Phase 9 — Encoding Fix Log

**Date:** 2026-05-01  
**Issue:** PowerShell 5.1 parsing error on Arabic strings in `.ps1` files  
**Error:** `The string is missing the terminator: ".` at `setup-rclone.ps1:33 char:75`

---

## Root Cause

PowerShell 5.1 on Windows can misparse Arabic RTL characters inside double-quoted
strings and here-strings, especially when the file is saved without BOM or with
mixed RTL/LTR content adjacent to escape sequences. The here-string in
`setup-rclone.ps1` contained Arabic followed by backtick-escaped quotes, causing
the tokenizer to lose track of string boundaries.
